# Codex Monitor 系统总体技术文档

## 1. 项目定位

Codex Monitor 是一个前端监控系统生产级 MVP，目标是让业务系统通过浏览器 SDK 接入后，完成前端错误、性能、接口、页面访问、点击行为和自定义事件的采集、上报、处理、存储、查询、展示、告警与 sourcemap 源码定位。

系统采用 monorepo 组织，包含四个核心运行单元：

- `packages/sdk`：业务方接入的浏览器监控 SDK。
- `apps/server`：NestJS API 服务，负责认证、项目、上报入口、查询、告警规则和 sourcemap 上传。
- `apps/worker`：BullMQ Worker，负责异步消费事件、清洗入库、聚合、告警和 sourcemap 反解。
- `apps/client`：React 管理控制台，供用户查看监控数据和管理配置。

## 2. 技术栈

| 层级 | 技术 |
| --- | --- |
| 包管理 | pnpm workspace |

## 4. 核心链路

### 4.1 数据采集链路

```text
业务页面
  ↓ 引入 SDK
SDK 自动采集错误、性能、行为、自定义事件
  ↓ 批量上报 POST /api/collect
Server 校验 appKey、域名、payload
  ↓ 写入 Redis/BullMQ 队列
Worker 消费 batch
  ↓ 清洗、分类、聚合、sourcemap 反解
MySQL 持久化
  ↓
Client 查询并展示
```

### 4.2 管理链路

```text
用户登录/注册
  ↓ JWT
创建项目
  ↓
创建应用并获得 appKey
  ↓
配置允许域名、release、告警规则、sourcemap
  ↓
业务系统接入 SDK
```

## 5. 模块职责

### SDK

- 初始化监控配置。
- 维护 sessionId、traceId、用户信息。
- 采集 JS 错误、Promise 错误、资源错误。
- 采集 Fetch/XHR 请求状态、耗时和失败。
- 采集 Web Vitals 指标。
- 采集 PV、路由变化、点击和自定义事件。
- 默认脱敏敏感字段。
- 使用 `sendBeacon` 或 `fetch` 批量上报。

### Server

- JWT 登录与注册。
- 项目和应用管理。
- SDK 上报入口。
- Redis 队列投递。
- 管理端事件查询。
- 总览看板聚合查询。
- 告警规则 CRUD。
- sourcemap 上传。

### Worker

- 监听 `monitor-events` 队列。
- 写入 `events_raw` 原始事件表。
- 按事件类型写入错误、性能、行为表。
- 写入分钟级聚合数据。
- 根据告警规则计算阈值并通知。
- 对错误堆栈进行 sourcemap 反解。

### Client

- 用户登录/注册。
- 项目和应用管理。
- SDK 接入代码展示。
- sourcemap 上传。
- 总览看板。
- 错误、性能、行为事件列表。
- 告警规则和触发记录展示。

## 6. 数据分层

| 表 | 说明 |
| --- | --- |
| `users` | 控制台用户 |
| `projects` | 用户拥有的项目 |
| `applications` | 项目下的业务应用，包含 appKey 和允许域名 |
| `api_keys` | SDK 上报 key |
| `events_raw` | 原始上报事件 |
| `error_events` | 错误事件明细 |
| `performance_events` | 性能和接口事件明细 |
| `behavior_events` | PV、点击、自定义事件明细 |
| `metric_aggregates` | 分钟级指标聚合 |
| `alert_rules` | 告警规则 |
| `alert_records` | 告警触发记录 |
| `sourcemap_artifacts` | sourcemap 文件记录 |

## 7. 安全设计

- 管理端接口使用 JWT。
- SDK 上报接口使用 `appKey`。
- 上报接口按应用 `allowedDomains` 校验来源域名。
- Server 限制 JSON payload 大小为 `1mb`。
- SDK 默认脱敏 query、token、password、cookie、authorization 等字段。
- 控制台项目按 `ownerId` 做隔离。
- sourcemap 上传接口要求登录，并校验应用归属。

## 8. 可扩展方向

当前是生产级 MVP，后续可扩展：

- ClickHouse 存储海量事件。
- Kafka 替换 Redis 队列。
- 复杂 RBAC 和团队协作。
- 多租户计费模型。
- 采样策略后台动态下发。
- 错误 issue 状态流转。
- SourceMap 按 bundle 文件精确匹配。
- Session Replay。
- OpenTelemetry trace 关联。
