# Codex Monitor 接口与数据模型文档

## 1. 认证方式

管理端接口使用 JWT：

```http
Authorization: Bearer <accessToken>
```

SDK 上报接口使用 payload 中的 `appKey`，不需要 JWT。

## 2. 认证接口

### POST /api/auth/register

请求：

```json
{
  "email": "user@example.com",
  "name": "用户",
  "password": "12345678"
}
```

响应：

```json
{
  "accessToken": "jwt-token",
  "user": {
    "id": "user-id",
    "email": "user@example.com",
    "name": "用户"
  }
}
```

### POST /api/auth/login

请求：

```json
{
  "email": "admin@example.com",
  "password": "admin123456"
}
```

响应同注册接口。

### GET /api/auth/me

返回当前 JWT 用户信息。

## 3. 项目与应用接口

### GET /api/projects

返回当前用户拥有的项目和应用列表。

### POST /api/projects

请求：

```json
{
  "name": "官网项目",
  "description": "官网前端监控"
}
```

### POST /api/apps

请求：

```json
{
  "projectId": "project-id",
  "name": "官网 Web",
  "environment": "production",
  "allowedDomains": ["example.com", "localhost"]
}
```

响应中包含 `appKey`。

## 4. SDK 上报接口

### POST /api/collect

请求：

```json
{
  "appKey": "app_xxx",
  "events": [
    {
      "type": "error",
      "name": "TypeError",
      "message": "Cannot read properties of undefined",
      "appKey": "app_xxx",
      "release": "1.0.0",
      "environment": "production",
      "sessionId": "session-id",
      "traceId": "trace-id",
      "url": "https://example.com/home",
      "userAgent": "Mozilla/5.0",
      "viewport": "1440x900",
      "timestamp": 1710000000000,
      "stack": "TypeError: ..."
    }
  ]
}
```

响应：

```json
{
  "accepted": 1
}
```

校验规则：

- 单次事件数量必须在 1 到 100 之间。
- envelope appKey 必须与事件 appKey 一致。
- appKey 必须存在。
- Origin 域名必须命中应用 `allowedDomains`。

## 5. 查询接口

### GET /api/dashboard/overview?applicationId=<id>

返回：

```json
{
  "pv": 1000,
  "uv": 300,
  "errors": 12,
  "errorRate": 0.012,
  "apiFailureRate": 0.03,
  "vitals": [
    { "name": "LCP", "average": 1800 }
  ],
  "topErrors": []
}
```

### GET /api/events/errors?applicationId=<id>

返回错误明细列表。

### GET /api/events/performance?applicationId=<id>

返回性能与接口明细列表。

### GET /api/events/behavior?applicationId=<id>

返回行为事件明细列表。

## 6. 告警接口

### GET /api/alerts/rules?applicationId=<id>

返回应用告警规则和最近触发记录。

### POST /api/alerts/rules

请求：

```json
{
  "applicationId": "app-id",
  "name": "错误数过高",
  "metric": "error_count",
  "operator": "gte",
  "threshold": 10,
  "durationMin": 5,
  "channel": "webhook",
  "target": "https://example.com/webhook"
}
```

### PUT /api/alerts/rules/:id

可更新：

- `name`
- `threshold`
- `enabled`
- `target`

### DELETE /api/alerts/rules/:id

删除告警规则。

## 7. Sourcemap 接口

### POST /api/sourcemaps/upload

`multipart/form-data` 字段：

- `applicationId`
- `release`
- `sourceRoot`
- `file`

上传成功后，Worker 会在处理同 applicationId、同 release 的错误事件时尝试反解 stack。

## 8. 主要数据模型

### User

控制台用户，包含邮箱、名称和密码哈希。

### Project

用户拥有的项目，是应用的上层隔离单元。

### Application

业务应用，包含 `appKey`、允许域名和默认环境。

### EventRaw

保存 SDK 原始事件，便于回溯和重新处理。

### ErrorEvent

保存错误明细，包含 fingerprint、stack、mappedStack、release、环境、页面 URL。

### PerformanceEvent

保存性能指标和接口请求指标。

### BehaviorEvent

保存 PV、点击和自定义事件。

### MetricAggregate

保存分钟级聚合指标，当前 metric 命名规则为 `<eventType>:<eventName>`。

### AlertRule / AlertRecord

保存告警规则和触发记录。

### SourcemapArtifact

保存 sourcemap 文件路径、release 和源码根路径。
