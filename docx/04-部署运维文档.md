# Codex Monitor 部署运维文档

## 1. 部署形态

当前项目提供 Docker Compose 部署，包含：

- `mysql`
- `redis`
- `server`
- `worker`
- `client`

生产环境建议将 MySQL、Redis 使用托管服务或独立高可用集群，应用服务使用容器平台或进程管理器部署。

## 2. Docker Compose 部署

```bash
cp .env.example .env
docker compose up --build -d
```

初始化数据库：

```bash
docker compose exec server pnpm prisma:migrate
docker compose exec server pnpm prisma:seed
```

查看日志：

```bash
docker compose logs -f server
docker compose logs -f worker
docker compose logs -f client
```

停止：

```bash
docker compose down
```

## 3. 本地开发部署

启动 MySQL 和 Redis 后：

```bash
pnpm install
pnpm db:generate
pnpm db:migrate
pnpm db:seed
pnpm dev
```

## 4. 生产环境建议

### Server

- 设置强随机 `JWT_SECRET`。
- 配置 HTTPS 反向代理。
- 限制 `/api/collect` 请求体大小。
- 开启进程守护和日志采集。
- 配置 API 限流。

### Worker

- Worker 可水平扩容。
- 多个 Worker 共享同一个 Redis 队列。
- 根据上报量调整 BullMQ concurrency。
- 监控 failed jobs 数量。

### MySQL

- `events_raw`、`error_events`、`performance_events`、`behavior_events` 数据增长快，需要归档策略。
- 建议按时间分区或定期冷热分层。
- 核心查询字段已建立索引：applicationId、type、name、occurredAt、fingerprint。

### Redis

- 用于队列，不建议和业务缓存混用。
- 配置持久化和内存上限。
- 关注队列堆积、失败任务、延迟。

## 5. 日志与监控

建议至少采集：

- Server 请求量、错误率、响应时间。
- Worker 消费速率、失败数量、队列堆积。
- MySQL 慢查询、连接数、磁盘使用。
- Redis 内存、连接数、队列长度。

## 6. 数据保留策略

推荐默认：

- `events_raw` 保留 7 到 15 天。
- 明细事件保留 30 到 90 天。
- 聚合数据保留 180 天以上。
- sourcemap 按 release 保留，与线上版本生命周期一致。

## 7. 备份恢复

必须备份：

- MySQL 数据库。
- sourcemap 上传目录 `uploads/sourcemaps`。
- `.env` 中生产配置。

恢复顺序：

1. 恢复 MySQL。
2. 恢复 sourcemap 文件目录。
3. 启动 Redis。
4. 启动 Server。
5. 启动 Worker。
6. 启动 Client。

## 8. 发布流程

推荐流程：

1. 合并代码。
2. 执行 `pnpm lint`、`pnpm test`、`pnpm build`。
3. 构建镜像。
4. 执行数据库 migration。
5. 滚动发布 Server 和 Worker。
6. 发布 Client。
7. 验证 SDK 上报和控制台查询。

## 9. 故障排查

### 上报接口返回 403

原因通常是应用允许域名未配置当前业务域名。

处理：

- 登录控制台查看应用允许域名。
- 确认浏览器请求的 Origin。

### Redis 队列堆积

处理：

- 增加 Worker 数量。
- 提高 Worker concurrency。
- 检查 MySQL 写入性能。
- 检查失败任务错误信息。

### MySQL 写入慢

处理：

- 检查索引。
- 检查磁盘和连接数。
- 减少原始事件保留时间。
- 后续可迁移明细数据到 ClickHouse。

### sourcemap 反解失败

处理：

- 确认 release 一致。
- 确认上传文件可读。
- 确认 stack 包含 line 和 column。
- 确认上传的是正确 bundle 对应的 map 文件。
