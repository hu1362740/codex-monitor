# Codex Monitor 详细使用文档

## 1. 环境要求

- Node.js 22 或更高版本。
- pnpm 9 或更高版本。
- MySQL 8。
- Redis 7。
- Docker 与 Docker Compose，推荐用于本地一键启动。

## 2. 安装依赖

```bash
corepack enable
pnpm install
```

如果本机没有 pnpm，推荐使用 Node 自带的 Corepack 启用。

## 3. 初始化环境变量

```bash
cp .env.example .env
```

关键环境变量：

| 变量 | 说明 |
| --- | --- |
| `DATABASE_URL` | Prisma 连接 MySQL 的地址 |
| `REDIS_HOST` | Redis 主机 |
| `REDIS_PORT` | Redis 端口 |
| `JWT_SECRET` | JWT 签名密钥，生产环境必须替换 |
| `SERVER_PORT` | Server 监听端口 |
| `VITE_API_BASE_URL` | Client 访问 Server 的 API 地址 |
| `SMTP_*` | 邮件告警配置 |

## 4. 初始化数据库

根脚本会通过 `scripts/run-with-env.mjs` 加载根目录 `.env`，无需在 `apps/server` 重复创建环境文件。

如果使用本机 MySQL 而不是 Docker Compose，请先使用管理员账号在 MySQL Workbench 中执行 `scripts/mysql-local-bootstrap.sql`，或将 `.env` 中的 `DATABASE_URL` 修改为已有本机数据库账号。

```bash
pnpm db:check
pnpm db:generate
pnpm db:migrate
pnpm db:seed
```

seed 会创建默认账号和示例应用：

- 邮箱：`admin@example.com`
- 密码：`admin123456`
- 示例 appKey：`demo-app-key`

## 5. 本地开发启动

```bash
pnpm dev
```

默认地址：

- 控制台：`http://localhost:5173`
- API：`http://localhost:3000/api`
- Redis：`localhost:6379`
- MySQL：`localhost:3306`

## 6. Docker Compose 启动

```bash
cp .env.example .env
docker compose up --build
```

首次启动后执行：

```bash
docker compose exec server pnpm prisma:migrate
docker compose exec server pnpm prisma:seed
```

## 7. 控制台使用流程

### 7.1 登录

访问 `http://localhost:5173`，使用默认账号登录，或注册新账号。

### 7.2 创建项目

进入“项目管理”，填写项目名称和描述。项目用于隔离不同业务线或产品。

### 7.3 创建应用

在项目下创建应用，填写：

- 项目 ID。
- 应用名称。
- 环境，例如 `production`、`staging`。
- 允许域名，例如 `localhost,example.com`。

创建后系统生成 `appKey`，业务系统接入 SDK 时必须使用该值。

### 7.4 应用配置

进入“应用配置”查看：

- appKey。
- SDK 接入代码。
- sourcemap 上传表单。

### 7.5 查看监控数据

菜单包含：

- 总览：PV、UV、错误数、错误率、接口失败率、Web Vitals、Top 错误。
- 错误监控：错误列表、页面 URL、版本、堆栈、sourcemap 反解结果。
- 性能监控：Web Vitals、接口耗时、状态码。
- 行为分析：PV、点击、自定义事件。
- 告警中心：规则配置和触发记录。

## 8. SDK 接入

安装：

```bash
pnpm add @codex-monitor/sdk
```

接入：

```ts
import { initMonitor, setUser, track, captureException } from "@codex-monitor/sdk";

initMonitor({
  appKey: "your-app-key",
  endpoint: "http://localhost:3000/api/collect",
  release: "1.0.0",
  environment: "production"
});

setUser({ id: "u_1", name: "张三" });
track("button_click", { name: "buy" });
captureException(new Error("manual error"));
```

## 9. 示例业务页

启动示例页：

```bash
pnpm --filter @codex-monitor/example-web dev
```

访问：

```text
http://localhost:5174
```

点击页面按钮可触发：

- 点击事件。
- 手动错误。
- 接口请求失败。

## 10. Sourcemap 使用

控制台上传：

1. 进入“应用配置”。
2. 选择应用。
3. 填写 release。
4. 上传 `.map` 文件。

脚本上传：

```bash
node scripts/upload-sourcemap.mjs http://localhost:3000/api <jwtToken> <applicationId> 1.0.0 ./dist/app.js.map webpack://src
```

SDK 上报错误时需要携带相同 `release`，Worker 才能找到对应 sourcemap。

## 11. 告警配置

支持指标：

- `error_count`：错误数。
- `error_rate`：错误率。
- `api_failure_rate`：接口失败率。
- `lcp`：LCP 平均值。

支持操作符：

- `gt`
- `gte`
- `lt`
- `lte`

支持渠道：

- `webhook`
- `email`

Webhook 会发送 JSON：

```json
{
  "title": "Codex Monitor 告警",
  "message": "规则名称: 当前值和阈值",
  "ruleId": "rule-id"
}
```

## 12. 常见问题

### 没有数据

检查：

- SDK `appKey` 是否正确。
- `endpoint` 是否指向 `/api/collect`。
- 应用允许域名是否包含当前页面域名。
- Server 和 Worker 是否都已启动。
- Redis 是否可用。

### 控制台登录失败

检查：

- 是否执行了 `pnpm db:seed`。
- `JWT_SECRET` 是否稳定。
- Server 是否连接到正确数据库。

### 错误没有源码定位

检查：

- SDK 的 `release` 是否与上传 sourcemap 的 release 一致。
- sourcemap 是否上传到对应 applicationId。
- Worker 是否运行。
- stack 中是否包含可解析的 `line:column`。
